Slide Line; (know in its similarities to “Four in a Square”) Mobile Strategy Game ________________________________________ Four in a Square MVP Scope — Cross-Platform Game (iOS / Android / PC) 1. Product Goal Deliver a modern, lightweight digital version of the classic strategy game “Four in a Square” (first to form a 2×2 block), replacing the old Flash version that previously ran on <http://Yet2play.com|Yet2play.com>. The MVP must be fast, stable, and fun, and ready for mobile app stores and desktop. ________________________________________ 2. Supported Platforms One shared codebase deployed to: • iOS (iPhone & iPad) • Android (phones & tablets) • PC (Windows desktop build or Web build) ________________________________________ 3. Core Gameplay (Must Have) Feature Description Board Square grid (default 8×8 or 10×10, configurable) Players 2 players Turns Players alternate placing one piece per turn Win Condition First player to form a 2×2 square of their own color wins immediately Draw If board fills without a 2×2 square Restart One-tap “Play Again” ________________________________________ 4. Game Modes (MVP) Required • Local 2-Player (“Pass & Play”) Recommended (still MVP-grade) • Single Player vs AI o One difficulty level (simple but competent) ________________________________________ 5. User Interface • Tap / click on a cell to place a piece • Clear indicator: “Player 1 / Player 2 Turn” • Winning 2×2 square visually highlighted • Responsive board scaling for: o phone portrait o phone landscape o tablet o PC screen ________________________________________ 6. Visual & Audio • Clean modern flat design • Two distinct colors for players • Simple piece-drop animation • Win animation + optional sound • Sound on/off toggle ________________________________________ 7. Technical Requirements • Built using a cross-platform engine (Unity or Godot preferred) • Must support: o Touch input o Mouse input • Runs offline (no internet required) • No login, no ads, no payments in MVP ________________________________________ 8. Deliverables Developer must provide: • ✔ Android build (APK or AAB) • ✔ iOS build (Xcode project or TestFlight-ready archive) • ✔ PC build (Windows exe or Web build) • ✔ Full source code • ✔ Build instructions ________________________________________ 9. Acceptance Criteria (How it is approved) The MVP is accepted when: • The game runs without crashes on all platforms • Win detection (2×2) works in all edge cases • UI works on small phones and large screens • Game can be restarted indefinitely • Builds are installable on real devices
